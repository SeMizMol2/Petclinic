# รายการแก้เอกสารอ้างอิงจากไฟล์ `.docx`

เอกสารอ้างอิง: `ระบบจัดการข้อมูลการรักษาและบริการสัตว์เลี้ยง กรณีศึกษาโรงพยาบาลสัตว์เมืองเลย (แก้ไข).docx`

ไฟล์นี้สรุปสิ่งที่ควรแก้ในเอกสารให้ตรงกับระบบปัจจุบันของโปรเจ็กต์

## สรุประบบปัจจุบัน

ฝั่งเจ้าของสัตว์เลี้ยงสามารถ:

- จัดการข้อมูลส่วนตัว
- จัดการข้อมูลสัตว์เลี้ยงของตนเอง
- ดูข้อมูลคลินิกและรายการบริการ
- ดูประวัติการรักษา
- ดูใบเสร็จและสถานะการชำระเงิน
- ดูนัดหมายที่คลินิกบันทึกไว้
- รับอีเมลแจ้งเตือนเมื่อคลินิกสร้าง/แก้ไข/ยกเลิกนัดหมาย

หมายเหตุสำคัญ:

- ไม่มีการจองนัดหมายจากฝั่ง user แล้ว
- การสร้างนัดหมายทำโดยฝั่ง admin/คลินิกเท่านั้น

## จุดที่ต้องแก้ในเอกสาร

### 1. ขอบเขตการศึกษา

ฝั่ง `เจ้าของสัตว์เลี้ยง` ควรระบุว่า:

- ดูข้อมูลการนัดหมายของตนเอง
- รับการแจ้งเตือนนัดหมายผ่านอีเมล

ไม่ควรระบุว่า:

- จองนัดหมายผ่านระบบ
- ส่งคำขอนัดหมายถึงคลินิก

### 2. DFD Level 1

Process การนัดหมายควรอธิบายเป็น:

- ผู้ดูแลระบบหรือคลินิกสร้างข้อมูลนัดหมาย
- ระบบบันทึกลง `tb_appointment`
- ระบบส่งอีเมลแจ้งเจ้าของสัตว์เลี้ยงเมื่อมีการสร้าง/แก้ไข/ยกเลิกนัดหมาย
- เจ้าของสัตว์เลี้ยงเข้ามาดูสถานะนัดหมายของตนเองได้

ไม่ควรอธิบายว่า user เป็นผู้สร้างคำขอนัดหมาย

### 3. ERD / Data Dictionary

สิ่งที่ควรยืนยันในเอกสาร:

- `tb_owner` ไม่ใช้ `owner_address`
- ตารางธุรกรรมทางการรักษาอ้างอิงสัตวแพทย์ด้วย `vet_id`
- `tb_treatment_detail` มีอยู่จริงและใช้เก็บหลายรายการบริการต่อการรักษา 1 ครั้ง

### 4. UI Design

ฝั่ง user:

- หน้านัดหมายเป็นหน้า “ดูข้อมูล”
- ไม่มีฟอร์มจองนัด
- ไม่มีปุ่ม `จองนัดหมายใหม่`

ฝั่ง admin:

- หน้านัดหมายเป็นหน้า “จัดการคิวนัดหมาย”
- สร้างนัดใหม่ได้จากฝั่งคลินิก
- แก้ไขเวลาและสถานะได้

## สถานะนัดหมาย

ระบบยังรองรับสถานะ:

- `รอ`
- `ยืนยัน`
- `ยกเลิก`

แต่ในบริบทธุรกิจปัจจุบัน `รอ` ใช้เป็นสถานะภายในได้ ไม่ได้หมายถึง “ผู้ใช้ส่งคำขอจองเอง” อีกต่อไป

## ข้อความแนะนำสำหรับใส่ในรายงาน

### ขอบเขตฝั่งเจ้าของสัตว์เลี้ยง

> เจ้าของสัตว์เลี้ยงสามารถเข้าสู่ระบบเพื่อจัดการข้อมูลส่วนตัว จัดการข้อมูลสัตว์เลี้ยงของตนเอง ตรวจสอบข้อมูลคลินิกและรายการบริการ ดูประวัติการรักษา ดูใบเสร็จและสถานะการชำระเงิน รวมถึงตรวจสอบข้อมูลนัดหมายที่คลินิกบันทึกไว้ได้ โดยระบบรองรับการแจ้งเตือนนัดหมายและการเปลี่ยนแปลงสถานะผ่านทางอีเมล

### Process การนัดหมาย

> กระบวนการจัดการข้อมูลการนัดหมายประกอบด้วยการที่ผู้ดูแลระบบหรือเจ้าหน้าที่คลินิกเป็นผู้สร้างและปรับปรุงข้อมูลนัดหมายของสัตว์เลี้ยงในระบบ โดยระบบจะบันทึกวัน เวลา เหตุผลการนัดหมาย และสถานะนัดหมาย พร้อมทั้งส่งการแจ้งเตือนผลการนัดหมายไปยังอีเมลของเจ้าของสัตว์เลี้ยง และเปิดให้เจ้าของสัตว์เลี้ยงตรวจสอบข้อมูลนัดหมายของตนเองได้ผ่านระบบ
